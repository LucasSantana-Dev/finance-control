## üéØ Core Design Principles

### 1. Clarity Over Cleverness
Every element has a clear purpose. Nothing decorative without function. Information hierarchy guides users naturally through their financial data.

### 2. Brazilian-First Localization
Currency (R$), dates (DD/MM/YYYY), language (Portuguese) feel native, not translated. Cultural context is understood and respected.

### 3. Visual Hierarchy
Most important information draws the eye naturally. Financial metrics, key actions, and critical alerts are immediately visible.

### 4. Responsive Grace
Beautiful on phone, tablet, desktop‚Äîadapts elegantly. Mobile-first approach ensures core functionality works perfectly on small screens.

### 5. Feedback Everywhere
Users always know what's happening, what they can do next. Every interaction provides immediate, clear feedback.

### 6. Trust Through Transparency
Clear data, obvious calculations, no hidden fees or surprises. Security indicators are visible but not intrusive.

### 7. Accessibility for All
WCAG 2.1 AA compliant. Screen reader friendly, keyboard navigable, color-blind accessible. Inclusive design benefits everyone.

### 8. Performance & Delight
Fast loading, smooth animations, instant feedback. Micro-interactions make the experience feel polished and responsive.

## ‚ôø Accessibility Requirements (WCAG 2.1 AA)

### Color Contrast
- **Text on Background**: Minimum 4.5:1 contrast ratio
- **Large Text (18px+)**: Minimum 3:1 contrast ratio
- **UI Components**: Minimum 3:1 contrast ratio
- **Non-text Elements**: Minimum 3:1 contrast ratio

### Keyboard Navigation
- All interactive elements must be keyboard accessible
- Focus indicators are clearly visible (sky blue outline, 2px solid)
- Logical tab order throughout the application
- Skip links for main content areas

### Screen Reader Support
- Semantic HTML5 elements (header, nav, main, article, section, footer)
- ARIA labels for interactive elements
- Alt text for all images and icons
- Descriptive link text (not "click here")
- Form labels properly associated with inputs

### Data Visualization Accessibility
- Text alternatives for all charts and graphs
- High contrast color schemes (not color-only indicators)
- Patterns/textures for color-blind users
- Keyboard navigation for interactive charts
- Screen reader descriptions of chart data

### Responsive Text
- Text resizable up to 200% without loss of functionality
- No horizontal scrolling at 200% zoom
- Content remains readable and usable

### Touch Targets
- Minimum 44x44px for all touch targets
- Adequate spacing between interactive elements (minimum 8px)
- Large hit areas for mobile interactions

## ‚ú® Micro-Interactions & Animations

### Principles
- **Purposeful**: Every animation serves a function
- **Subtle**: Enhance, don't distract
- **Fast**: 200-300ms for micro-interactions, 300-500ms for transitions
- **Natural**: Easing functions that feel organic (ease-out, ease-in-out)

### Loading States
- **Skeleton Loaders**: Match final content structure
- **Progress Indicators**: For long operations (data export, large data loads)
- **Spinner Animations**: For quick operations (form submission, API calls)
- **Pulse Effects**: For real-time data updates

### Button Interactions
- **Hover**: Slight lift (shadow deepens), color brightens 10%
- **Active/Press**: Slight scale down (0.98), shadow reduces
- **Loading**: Spinner replaces text, button disabled
- **Success**: Checkmark animation, brief color change to green
- **Error**: Shake animation, red border highlight

### Form Interactions
- **Focus**: Sky blue border glow, label moves up (if floating label)
- **Validation**: Real-time feedback, helpful error messages
- **Success**: Green checkmark, subtle success animation
- **Error**: Red border, shake animation, error message below field

### Card Interactions
- **Hover**: Subtle lift (shadow deepens), cursor pointer
- **Click**: Brief scale down, then navigation
- **Loading**: Skeleton placeholder matching card structure

### Data Visualization Interactions
- **Hover**: Tooltip appears, data point highlights
- **Click**: Filters applied, smooth transition
- **Animation**: Data loads with smooth transitions, not instant

### Page Transitions
- **Route Changes**: Fade or slide transitions (300ms)
- **Modal Open**: Backdrop fades in, modal slides up (300ms)
- **Modal Close**: Reverse animation, backdrop fades out

### Success/Error Feedback
- **Toast Notifications**: Slide in from top-right, auto-dismiss after 3-5 seconds
- **Inline Messages**: Appear below relevant element, persist until dismissed
- **Confirmation Dialogs**: Smooth fade-in, clear actions

## üìä Data Visualization Guidelines

### Chart Types

#### Line Charts
- **Usage**: Trends over time (income, expenses, net worth)
- **Colors**: Green for income, red for expenses, sky blue for net worth
- **Interactivity**: Hover for values, click to filter
- **Accessibility**: High contrast, patterns for color-blind users

#### Donut/Pie Charts
- **Usage**: Category breakdowns, spending distribution
- **Colors**: Distinct, accessible palette (not just color)
- **Interactivity**: Hover highlights segment, click to filter
- **Accessibility**: Patterns/textures, text labels, legend

#### Bar Charts
- **Usage**: Comparisons (monthly comparisons, category comparisons)
- **Colors**: Sky blue primary, semantic colors for meaning
- **Interactivity**: Hover for details, click for drill-down
- **Accessibility**: Clear labels, high contrast

### Chart Best Practices
- **Clear Titles**: Descriptive, contextual titles
- **Axis Labels**: Clear, formatted correctly (currency, dates)
- **Legends**: Visible, interactive, accessible
- **Tooltips**: Informative, well-formatted, accessible
- **Empty States**: Friendly message, guidance on adding data
- **Loading States**: Skeleton loaders matching chart structure

### Accessibility for Charts
- **Text Alternatives**: Descriptive summaries of chart data
- **Keyboard Navigation**: Arrow keys to navigate data points
- **Screen Reader Support**: ARIA labels, data tables as fallback
- **Color Independence**: Not relying solely on color to convey meaning

## üöÄ Performance Considerations

### Loading Performance
- **Initial Load**: < 3 seconds on 3G connection
- **Time to Interactive**: < 5 seconds
- **Lazy Loading**: Images, charts, below-fold content
- **Code Splitting**: Route-based code splitting
- **Caching**: Aggressive caching for static assets

### Runtime Performance
- **Smooth Animations**: 60fps, use CSS transforms
- **Debounced Inputs**: Search, filters, form validation
- **Virtual Scrolling**: For long lists (transactions, goals)
- **Optimistic Updates**: Immediate UI feedback, sync in background

### Data Fetching
- **Skeleton Loaders**: Show structure while loading
- **Progressive Loading**: Critical data first, details later
- **Error Handling**: Graceful degradation, retry mechanisms
- **Caching Strategy**: Cache dashboard data, invalidate appropriately
